# EVIS — Checklist Obra Piloto

> Uso: durante ou imediatamente após cada etapa da operação real.
> Responda com: ✅ fluiu / ⚠ travou / ❌ quebrou + observação livre.
> Não filtre. O que pareceu pequeno provavelmente é o mais importante.

---

## 1. Entrada do Lead / Oportunidade

**Cadastro**
- [ ] O cadastro da oportunidade foi rápido?
- [ ] Os campos pedidos faziam sentido para o momento?
- [ ] Ficou alguma informação importante sem campo?
- [ ] Em algum momento o operador precisou sair do sistema (WhatsApp, planilha, anotação no papel)?

**Clareza de status**
- [ ] Ficou claro em que etapa a oportunidade estava?
- [ ] O operador soube o que fazer depois do cadastro?
- [ ] A lista de oportunidades ficou fácil de ler?

## 2. Upload de Documentos

**Processo de upload**

**Organização**
- [ ] Os arquivos ficaram organizados de forma compreensível?
- [ ] O operador conseguiu encontrar um arquivo já enviado rapidamente?
- [ ] Houve confusão sobre qual versão do arquivo era a mais recente?

## 3. Orçamentista IA

**Leitura dos documentos**
- [ ] A IA extraiu os itens corretamente?
- [ ] Houve itens claramente errados ou inventados (hallucination)?
- [ ] A IA sinalizou corretamente quando não tinha certeza?
- [ ] O operador confiou no resultado ou precisou verificar item por item?

**Fluxo HITL (revisão humana)**
- [ ] O operador entendeu o que estava sendo pedido em cada ponto de revisão?
- [ ] A quantidade de aprovações necessárias foi razoável?
- [ ] Alguma aprovação pareceu desnecessária ou burocrática?
- [ ] Em algum momento o operador aprovou sem ler (fadiga de clique)?
- [ ] O resultado final do orçamento refletiu as decisões tomadas?

**Confiança**
- [ ] O operador apresentaria esse orçamento a um cliente sem edição manual?
- [ ] Se não: qual foi o principal motivo?

## 7. Sensação Operacional

> Esta seção vale mais que todas as anteriores juntas.
> Responda com honestidade brutal.

**Fricção**
- [ ] Em algum momento você sentiu que estava lutando contra o sistema?
- [ ] Em algum momento você perdeu confiança no que o sistema mostrava?
- [ ] Em algum momento você quis voltar para WhatsApp, planilha ou papel?
- [ ] Em algum momento o sistema pareceu trabalhar contra você?

**Carga cognitiva**
- [ ] Em algum momento você teve que pensar demais para fazer algo simples?
- [ ] Em algum momento você não soube o que estava acontecendo?
- [ ] Em algum momento você não soube o que fazer a seguir?

**Confiança no sistema**
- [ ] Você indicaria esse sistema para um engenheiro de obra hoje?
- [ ] Se não: o que precisaria mudar primeiro?
- [ ] Em qual etapa o sistema gerou mais valor real?
- [ ] Em qual etapa o sistema gerou mais atrito?

**A pergunta mais importante:**
> "Se você tivesse que usar isso amanhã numa obra de verdade, o que te impediria?"

## Meta do Piloto

O objetivo **não é** que tudo funcione perfeitamente.

O objetivo é descobrir:
1. **Onde o operador trava** — para corrigir antes de escalar
2. **Onde o sistema gera confiança real** — para amplificar
3. **O que o operador faz naturalmente** — para o sistema se adaptar, não o contrário

Um piloto com 3 travamentos críticos descobertos vale mais que 3 meses de engenharia sem feedback real.
